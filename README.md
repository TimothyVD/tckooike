# TC Kooike — Website & Competition Scheduler

This repository contains two things:

1. **The TC Kooike club website** — a static site served via GitHub Pages (`docs/`). All content is written in Markdown files under `input/` and built into `docs/index.html` automatically by GitHub Actions.
2. **A competition scheduler** — uses constraint programming (CP-SAT) to generate an optimal poule-format match schedule from team availabilities and terrain slots.

## Editing website content

All website content lives in `input/*.md`. Edit a file, push, and GitHub Actions rebuilds the site automatically within ~1 minute. No manual script run needed.

### Markdown syntax supported

| Syntax | Result |
|---|---|
| `## Heading` | Bold coloured section heading |
| `### Subheading` | Smaller bold heading |
| `**bold**` | Bold text |
| `[label](url)` | Link (opens in new tab; `mailto:` and `tel:` stay in same tab) |
| `- item` | Bullet list |
| `\| Col \| Col \|` | Table |
| `> note text` | Muted small help paragraph |
| `? Question` + answer on next line | FAQ block |
| `[cta: Button label](url)` | Call-to-action button |
| `// comment` | Ignored — use anywhere to leave notes or disable a line |

### File-specific formats

#### `input/kalender.md` — calendar events
```markdown
## DD maand YYYY | Event title
Optional note shown below the title
hide-after: YYYY-MM-DD    ← hides the note after this date (event itself stays)
```

#### `input/interclub.md` — interclub matches
```markdown
## TYPE | Kapitein Naam | Reeks
- DD/MM/YYYY HH:MM | Ontvangende Club | Bezoekende Club
```

#### `input/sponsors.md` — sponsors
```markdown
- Naam | images/sponsors/logo.png | https://website.com
- Naam | images/sponsors/logo.png          ← url is optional
```

#### `input/bestuur.md` — board members
```markdown
- Naam | Rol | images/bestuur/foto.jpg
- Naam | Rol                               ← photo is optional
```

#### `input/sfeer.md` — sfeer photo gallery
```markdown
## Section caption
- filename                      ← references docs/images/sfeer/filename.jpg
- filename | center 65%         ← optional CSS object-position for cropping
```

## Adding a new sfeer photo

1. Copy the full-size `.jpg` to `docs/images/sfeer/`
2. Add `- filename` (without extension) to `input/sfeer.md` under the right section
3. Push — GitHub Actions generates the thumbnail automatically and commits it

To preview locally first: run `python build_site.py` — the thumbnail is generated immediately.

## Running the competition scheduler

The scheduler is run locally to compute a new match schedule. It outputs `input/schedule.json`, which is then picked up by the site builder.

### Installation

```bash
pip install -r requirements.txt
pip install Pillow          # only needed for thumbnail generation in build_site.py
```

### Built-in demo

```bash
python competition_scheduler.py
```

Runs a synthetic 2-poule demo, saves `input/schedule.json`, and writes a local `schedule_demo.html` preview.

### From CSV files

```bash
python competition_scheduler.py \
  --teams  input/example_teams.csv \
  --avail  input/example_team_availabilities.csv \
  --slots  input/example_terrain_slots.csv \
  --output output/schedule.xlsx
```

| Argument | Description |
|---|---|
| `--teams` | CSV with team names and poule assignments |
| `--avail` | CSV with team availabilities (Google Forms export) |
| `--slots` | CSV with terrain slot definitions |
| `--output` | Output Excel file |
| `--timelimit` | Solver time limit in seconds (default: 60) |

After running, push `input/schedule.json` — GitHub Actions will publish the updated schedule to the live site.

## Building the site locally

```bash
python build_site.py
```

Reads all `input/*.md` files and `input/schedule.json`, generates any missing sfeer thumbnails, and writes `docs/index.html`. Open it in a browser to preview changes before pushing.

---

## GitHub Actions

The workflow in `.github/workflows/build.yml` triggers automatically when any file under `input/` or a new `.jpg` in `docs/images/sfeer/` is pushed. It:

1. Runs `python build_site.py`
2. Generates any missing sfeer thumbnails
3. Commits and pushes the updated `docs/index.html` and new thumbnails

Only `site_builder.py` and `build_site.py` are used in CI — `competition_scheduler.py` and its heavy dependencies (ortools, pandas) are never needed there.

## Input file formats (scheduler)

### `input/example_teams.csv`

| Column | Required | Description |
|---|---|---|
| `Team` | yes | Team name (must match availability CSV) |
| `Poule` | no | Poule identifier (defaults to `A`) |
| `player_1` | no | Name of first player |
| `player_2` | no | Name of second player |
| `tel_player_1` | no | Phone number of player 1 |
| `tel_player_2` | no | Phone number of player 2 |

### `input/example_team_availabilities.csv`

**Format A — one column per slot (Google Forms checkbox grid):**
```
Team,2026-05-04 18:00,2026-05-04 19:30,...
Alpha,TRUE,,TRUE,...
```

**Format B — comma-separated list:**
```
Team,Available slots
Alpha,"2026-05-04 18:00, 2026-05-11 18:00"
```

### `input/example_terrain_slots.csv`

```
date,time,terrain_id
2026-05-04,18:00,1
2026-05-04,18:00,2
2026-05-04,19:30,1
```
